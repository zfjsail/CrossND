================================================================================
核心文件说明
================================================================================

1. environment.py (RL环境定义)
   ────────────────────────────────────────────────────────────────────────
   类:
   - CrossNDRLEnvironment: 多轮分类任务环境
     * 初始化和重置环境
     * 环境步骤执行
     * 准确率计算 (单轮、逐轮、汇总)
     * Episode数据管理
   
   - BatchedRLEnvironment: 批量环境处理
     * 支持批量重置和步骤执行
     * 批量观察管理
   
   主要方法:
   - reset(episode_id): 重置环境
   - step(predictions, logits): 执行步骤
   - compute_episode_accuracy(predictions, labels): 计算准确率
   - compute_per_turn_accuracy(predictions, labels): 逐轮准确率

2. reward.py (Reward函数)
   ────────────────────────────────────────────────────────────────────────
   类:
   - CrossNDRewardFunction: 多种reward计算方式
     * 支持3种reward类型: accuracy, f1, balanced_acc
     * Outcome reward (最终结果)
     * Turn-level reward (逐轮改进)
     * Advantage estimation
   
   - TurnLevelRewardShaper: Reward shaping和normalization
     * Discount操作
     * Normalization处理
     * Clipping限制
   
   主要方法:
   - compute_outcome_reward(): 最终reward
   - compute_turn_level_rewards(): 逐轮reward
   - compute_advantages(): advantage计算
   - compute_combined_reward(): 综合reward
   - shape_rewards(): reward shaping

3. agent.py (RL Agent)
   ────────────────────────────────────────────────────────────────────────
   类:
   - CrossNDAgent: 与环境交互的智能体
     * Episode管理
     * 动作预测 (贪心/采样)
     * 轨迹生成和记录
   
   - MultiAgentCollector: 批量Agent管理
     * 并行轨迹收集
     * Group reward计算
   
   数据结构:
   - AgentAction: 单个动作 (标签、置信度、logits)
   - AgentStep: 单步执行 (轮次、观察、动作、reward)
   - AgentTrajectory: 完整episode轨迹
   
   主要方法:
   - initialize_episode(): 初始化episode
   - predict_action(): 预测动作
   - step(): 执行一步
   - generate_trajectory(): 生成完整轨迹
   - compute_trajectory_logprobs(): 计算log概率

4. train_rl.py (GRPO训练脚本)
   ────────────────────────────────────────────────────────────────────────
   类:
   - GRPOConfig: 训练配置
     * 学习率、epoch数、batch大小
     * GRPO特定参数 (num_generations, turn_advantage_coef)
     * Reward权重设置
     * 输出和日志配置
   
   - GRPOTrainer: GRPO算法实现
     * 完整训练循环
     * Epoch训练
     * Loss计算
     * 模型评估和保存
   
   主要方法:
   - train(): 完整训练
   - _train_epoch(): 单个epoch
   - _compute_grpo_loss(): GRPO loss
   - _evaluate(): 评估模型
   - _save_checkpoint(): 保存检查点

5. __init__.py (模块初始化)
   ────────────────────────────────────────────────────────────────────────
   导出所有主要类和函数，便于模块导入:
   from rl import CrossNDRLEnvironment, CrossNDRewardFunction, ...

================================================================================
性能预期
================================================================================

内存和计算:
- 相比SL增加内存: 1-2倍 (取决于num_generations)
- 相比SL增加时间: num_generations倍的forward pass
- 推荐GPU: 16GB+ VRAM (batch_size=4-8)

性能提升:
- Author-level AUC: 通常提升 2-5%
- Author-level MAP: 通常提升 3-7%
- 在不平衡数据上效果更显著

成本vs收益:
- 快速迭代: 1-2小时额外训练时间，~1-2%提升
- 完整优化: 6-12小时额外训练时间，~3-5%提升

================================================================================
关键设计决策
================================================================================

1. 两层次Reward设计
   - Outcome Reward: 衡量最终结果
   - Turn-level Reward: 衡量每一轮改进
   → 鼓励逐步改进而不是只关注最终结果

2. Advantage Estimation
   - Moving Average Baseline: 历史reward的EMA
   - 自动更新，无需手动维护
   → 减少variance，加速收敛

3. GRPO算法
   - Group Relative Policy Optimization
   - 在group内计算相对reward
   → 更稳定的训练

4. 模块化设计
   - 环境、Reward、Agent、训练分离
   - 清晰的接口定义
   → 易于扩展和测试
